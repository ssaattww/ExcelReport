# Codex委譲パターン

## PMがやること
- タスク作成・管理（TaskCreate/TaskUpdate）
- tasks/*.md の更新
- Codexへの指示出し（小分けに、丁寧に）
- Codex出力の最終判断（ただしコードを直接読むのではなくCodexレビュー結果で判断）
- git操作（ブランチ作成、コミット）
- テスト実行（Codex sandboxの制約がある場合のみ）

## Codexがやること
- ソースコード調査 → reports/ にレポート出力
- コードレビュー → reports/ にレビュー結果出力
- 実装（TDD先行）
- ビルドエラー修正
- テスト失敗修正

## Codex指示のコツ
- git diff をレビューさせる時は `-- ':!**/obj/**'` でobjを除外
- 一度に渡すタスクは1-2件まで
- 具体的なファイルパスと行番号を示す
- 期待する出力形式（レポート先）を明記
