# issue #58 設計レビュー: 挿入サイズ不一致と設計書命名修正 (2026-04-12)

## 背景
- ユーザー懸念: 挿入元コンポーネント行数が挿入先の想定定義範囲より大きい場合に、書式崩れが起きる可能性がある。
- 追加要望: `Issue58` だけでは設計書の目的が分かりづらいため、設計書名を明確化する。

## 確認結果（修正前）
- 既存設計には `use` 展開時の行シフト方針はあったが、
  - 挿入先想定範囲を超過した行の扱い
  - overflow発生時のWarning仕様
  - 境界をまたぐ結合セルの停止条件
  が明示されておらず、サイズ不一致時の書式保全要件として不十分だった。

## 実施した修正
1. 設計書タイトルを明確化
- `Issue #58 詳細設計（承認依頼版）`
- -> `Excelテンプレート対応 詳細設計（Issue #58 / 承認依頼版）`

2. 設計対象を明記
- ドキュメント冒頭に `Excel -> XML Template -> DSL` を対象とする旨を追加。

3. サイズ不一致ケースの仕様を追加
- `10.9 サイズ不一致時の挿入ルール（挿入元 > 挿入先想定範囲）` を新設。
- 追加内容:
  - 挿入先定義範囲はクリップ境界ではなくアンカー扱い
  - 挿入元実高さまで展開（切り詰め禁止）
  - overflow行は挿入元書式ルールを適用
  - 後続行は既存値/書式/結合状態を保って相対シフト
  - `TemplateRangeOverflow` Warning を座標付きで記録
  - 結合セル境界破壊は Error 停止

4. テスト観点と合意表の補強
- 11.5 に overflow時の書式保持/Warning検証ケースを追加。
- 12.2 に「挿入元高さ > 挿入先想定行数」の合意項目を追加。

5. ファイル名と配置先を更新
- `Design/Issue58/Issue58_DetailDesign.md`
- -> `Design/ExcelTemplate/ExcelTemplate_DetailDesign.md`
