## 2. Status Legend

- `Ready`: explicit and usable now; consistent with Phase 2 baseline.
- `Partial`: implemented in part, but missing standardization or binding details.
- `Gap`: missing explicit definition or missing required structure.
- `N/A`: not directly applicable to the skill role (utility-only scope).

## 3. Dimension Definitions and Pass Criteria

| Dimension | Definition | Pass Criteria (`Ready`) |
|---|---|---|
| Execution contract compliance | Input/output/status schema binding | Skill explicitly binds to required contract fields and status semantics (`completed/needs_input/blocked/failed`) with actionable handling |
| Stop/approval protocol integration | `[Stop: ...]` / `[Approve: ...]` tags and resume constraints | Skill defines stop triggers, approval gate tags, and resume requirements (including approval truth condition) |
| Quality gate output format | Standardized quality gate structure (`result/evidence/blockers`) | Skill requires a repeatable quality gate output shape and fail/pass branching evidence |
| Sandbox mode alignment | Declared sandbox assumptions and escalation boundaries | Skill defines sandbox decision source, default mode, and escalation constraints |
| Reference synchronization | Required reference set and path consistency | Skill cites required reference files and defines drift resolution/source-of-truth behavior |
